Política de Privacidad - Aplicación Chasquido

Desarrollador: Benavides Ciurlizza, Oscar Javier
Fecha de última actualización: 13 de agosto de 2025

1. Introducción

Esta Política de Privacidad describe cómo la aplicación Chasquido, diseñada para la gestión de reparto y distribución, recopila, usa y protege la información de los usuarios. Al utilizar nuestra aplicación, aceptas los términos descritos en esta política.

2. Información que Recopilamos

2.1 Datos de Autenticación

  • Credenciales de usuario: Nombre de usuario y contraseña para el acceso tradicional.
  • Número de teléfono: Para el sistema de autenticación mediante código de verificación vía WhatsApp.
  • Códigos de verificación: Códigos temporales enviados por WhatsApp para validar el acceso.
  • Datos de sesión de reparto: Información sobre la placa del vehículo y fecha de reparto seleccionada.

2.2 Datos de Ubicación

  • Coordenadas GPS: Recopilamos la ubicación geográfica de tu dispositivo (latitud y longitud) cada 5 minutos durante el proceso de tracking activo.
  • Activación del tracking: Los datos de ubicación solo se capturan cuando el usuario inicia manualmente el proceso de tracking desde la pantalla de inicio.
  • Transmisión: Estas coordenadas se envían automáticamente a nuestros servidores para seguimiento de rutas de reparto.

2.3 Datos de Actividad de Reparto

  • Estados de pedidos: Registros de pedidos entregados, rechazados y refacturados.
  • Información de pedidos: Datos relacionados con los pedidos asignados, incluyendo direcciones de entrega y estados de paquetes.
  • Refacturaciones: Datos editados por el usuario durante el proceso de refacturación de pedidos.
  • Tokens de notificaciones: Identificadores únicos del dispositivo para el envío de notificaciones push.

2.4 Firebase y Servicios de Google

  • Firebase Cloud Messaging: Utilizamos Firebase para el envío de notificaciones push sobre refacturaciones, lo cual requiere:
    • Token único del dispositivo para identificar dónde enviar las notificaciones
    • ID de publicidad de Android (requerido por Firebase)
    • Información básica del dispositivo para compatibilidad
  • Firebase Analytics (básico): Firebase incluye automáticamente analytics básicos que nos ayudan a:
    • Detectar errores y problemas técnicos
    • Verificar que las notificaciones se entreguen correctamente
    • Monitorear el rendimiento básico de la aplicación
  • Propósito principal: El uso de Firebase se centra en garantizar que recibas las notificaciones de refacturaciones de manera confiable y oportuna.
  • Datos mínimos: Solo recopilamos los datos mínimos necesarios para que el sistema de notificaciones funcione correctamente.

2.5 Datos que NO Recopilamos

  • No accedemos a contactos: La aplicación no lee ni almacena información de tu agenda de contactos.
  • No accedemos a archivos personales: No leemos documentos, fotos o archivos personales del dispositivo.
  • No recopilamos datos sensibles: No accedemos a información bancaria, contraseñas de otras aplicaciones o datos médicos.

3. Cómo Utilizamos la Información

Los datos recopilados se utilizan exclusivamente para:

  • Autenticación de usuarios: Verificar la identidad de los repartidores y validar el acceso al sistema.
  • Gestión de sesiones de reparto: Asignar al usuario a la placa y fecha de reparto correspondiente.
  • Seguimiento de rutas: Monitorear las rutas de entrega para optimización logística y control operativo.
  • Gestión de pedidos: Facilitar la visualización, actualización y gestión de estados de pedidos.
  • Procesamiento de refacturaciones: Permitir la edición y creación de nuevos pedidos basados en modificaciones del usuario.
  • Notificaciones de refacturaciones: Enviar alertas cuando se crean nuevas refacturaciones con el usuario y cuando cambian de estado (éxito o error).
  • Análisis operativo: Generar reportes sobre eficiencia de entregas, rechazos y actividad de reparto.
  • Mantenimiento técnico: Detectar y corregir errores de la aplicación para garantizar su funcionamiento.
  • Optimización de notificaciones: Asegurar que las notificaciones push se entreguen de manera confiable.

4. Compartición de Datos

4.1 Acceso Interno

  • Los datos se integran y comparten dentro de nuestro sistema ERP empresarial para gestión logística.
  • Personal autorizado: Supervisores de distribución, gerentes logísticos y personal administrativo tienen acceso para fines operativos.
  • Propósito específico: El acceso se limita a funciones relacionadas con la gestión de reparto, control de entregas y optimización logística.

4.2 Servicios de Terceros

  • Google/Firebase: Compartimos datos mínimos con Firebase (Google) exclusivamente para el funcionamiento del sistema de notificaciones push y monitoreo básico de errores.
  • Servicio de WhatsApp: Compartimos números de teléfono con la plataforma WhatsApp únicamente para envío de códigos de verificación.
  • No vendemos datos: No vendemos, alquilamos ni compartimos tus datos personales con fines comerciales o publicitarios.

4.3 Requerimientos Legales

  • Solo compartiremos información si es requerido por autoridades legales competentes o para cumplir con obligaciones legales.

5. Seguridad de los Datos

Implementamos medidas de seguridad robustas para proteger tu información:

  • Cifrado en tránsito: Todas las comunicaciones utilizan protocolo HTTPS para cifrar datos durante la transmisión.
  • Autenticación multi-factor: Sistema de verificación por WhatsApp y credenciales para acceso seguro.
  • Almacenamiento seguro: Los datos se almacenan en servidores en la nube con medidas de seguridad estándar de la industria.
  • Acceso controlado: Solo personal autorizado y sistemas autenticados pueden acceder a los datos.
  • Monitoreo de seguridad: Supervisión continua de accesos y actividades sospechosas.
  • Protección de Firebase: Los datos analíticos están protegidos por las medidas de seguridad de Google Cloud Platform.

6. Control de Privacidad y Derechos del Usuario

Como usuario de Chasquido, tienes derecho a:

  • Acceso: Solicitar una copia de todos los datos que tenemos sobre ti.
  • Rectificación: Corregir datos inexactos o incompletos a través de la función de actualización de usuario.
  • Eliminación: Solicitar la eliminación de tus datos personales (sujeto a obligaciones legales y operativas).
  • Restricción: Limitar el procesamiento de tus datos en ciertas circunstancias.
  • Portabilidad: Obtener tus datos en un formato estructurado y legible.
  • Control de tracking: Iniciar y detener el seguimiento de ubicación desde la aplicación.
  • Opt-out de analytics: Puedes desactivar el seguimiento de analytics en la configuración de tu dispositivo Android (Configuración > Google > Anuncios > Restablecer ID de publicidad).

Para ejercer estos derechos, contacta con nosotros en: oscartrabajoremoto@gmail.com

7. Retención de Datos

  • Datos de ubicación: Se conservan durante el período necesario para análisis operativo y cumplimiento de obligaciones logísticas.
  • Datos de autenticación: Se mantienen mientras la cuenta esté activa y por un período adicional por motivos de seguridad.
  • Historial de pedidos: Se conserva para fines de auditoría, análisis operativo y resolución de disputas.
  • Datos de analytics: Los datos de Firebase Analytics se conservan según las políticas de retención de Google (por defecto 26 meses).
  • Eliminación automática: Los datos se eliminan cuando ya no sean necesarios para los fines operativos o legales.

8. Permisos de la Aplicación

La aplicación Chasquido solicita los siguientes permisos:

  • Ubicación: Necesario para el tracking de rutas durante el proceso de reparto.
  • Acceso a red: Para sincronizar datos con el sistema ERP y recibir actualizaciones de pedidos.
  • Notificaciones: Para recibir alertas específicas sobre refacturaciones (creación y cambios de estado).
  • WhatsApp: Para recibir códigos de verificación durante el proceso de autenticación.
  • Almacenamiento: Para guardar configuraciones de usuario y datos temporales de la aplicación.
  • ID de Publicidad (Android 13+): Para analytics y mejora de la aplicación (opcional en Android 13+).

Puedes gestionar estos permisos en la configuración de tu dispositivo. Ten en cuenta que revocar ciertos permisos puede afectar la funcionalidad de la aplicación.

9. Notificaciones Push

Chasquido utiliza notificaciones push exclusivamente para refacturaciones:

  • Nueva refacturación creada: Recibirás una notificación cuando se cree una nueva refacturación asociada a tu usuario.
  • Cambio de estado de refacturación: Alertas cuando una refacturación cambie de estado, ya sea que se procese correctamente o si ocurre algún error.
  • Control de usuario: Puedes desactivar las notificaciones en la configuración de tu dispositivo sin afectar otras funciones de la app.

10. Cambios en la Política de Privacidad

Nos reservamos el derecho de actualizar esta política para reflejar cambios en nuestras prácticas o requerimientos legales. Los cambios se notificarán a través de:

  • Notificación push dentro de la aplicación
  • Correo electrónico (si está disponible en tu perfil)
  • Actualización de la fecha en esta política
